# App UI Spec

## Purpose

定義 `app.py` 的 Gradio 使用者介面規格，包含三個主要 tab、繁體中文標籤、UI 與 backend function 的介接方式，以及 optional image generation 的前端行為。

## Responsibilities

- 建立 Gradio App entry point。
- 提供三個 tabs：烏龜照護問答、龜缸環境診斷、龜缸設計與圖片生成。
- 收集使用者輸入並轉交 backend modules。
- 顯示 LLM 回答、診斷報告、風險摘要、RAG references、設計 prompt、圖片生成狀態與 optional image。
- 確保沒有 API key 或圖片生成不可用時仍可 demo。

## Data Format

UI handlers should normalize component values into dictionaries before calling backend functions.

Example design input:

```python
{
    "species": species,
    "tank_length_cm": tank_length,
    "tank_width_cm": tank_width,
    "tank_height_cm": tank_height,
    "water_depth_cm": water_depth,
    "desired_style": desired_style,
    "required_elements": required_elements,
    "elements_to_avoid": elements_to_avoid,
    "user_wants_image": generate_image_choice == "是",
}
```

## Error / Fallback Behavior

- If backend returns an error message, display it as readable text instead of crashing the UI.
- If mock mode is active, display mock text and retrieved references.
- If image generation is disabled, show: `圖片生成功能未啟用。目前僅顯示圖片生成 Prompt。`
- If image generation fails, keep prompt-only outputs visible and display the failure status.
- If image path is `None`, leave image output empty.

## Safety Requirements

- UI text must not imply veterinary diagnosis.
- Health-related outputs must display safety warnings when generated by backend.
- Tank design tab must remind that generated images are visualization only, not engineering or veterinary recommendations.
